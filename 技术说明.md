# 相机标定辅助工具 — 技术说明

## 概述

基于 RTSP 协议连接 IP 摄像机，自动检测棋盘格标定板，采集高质量标定图片的工具。支持多品牌相机（海康威视/大华/宇视），含 OSD 控制、语音播报、图片质量检测、姿态去重和重投影误差筛选。

## 架构

```
main.py                  GUI 主入口
├── rtsp_capture.py       RTSP 流捕获 & 截图
├── chessboard_detector.py  棋盘格角点检测 & 质量检测
├── voice.py              TTS 语音播报
├── osd_control.py        OSD 文字叠加控制（ONVIF + HTTP）
├── screenshots/
│   ├── config.json        UI 配置持久化
│   ├── rtsp_history.json  RTSP 地址历史
│   └── {序列号}/           按序列号归档的截图
```

## 模块说明

### main.py — GUI 主程序

- **App 类**：Tkinter 主窗口，1280×800
- **布局**：顶部控制面板 → 状态栏 → 日志区 → 双栏预览区（实时画面 / 抓图结果） | 右侧照片列表
- **截图循环**：后台线程按间隔采集帧 → 棋盘格检测 → 4 道质量检测 → 姿态去重 → 保存
- **照片管理**：按序列号建文件夹保存，启动/切换序列号/开始截图时均加载文件夹全量照片
- **重投影筛选**：本次新增截图达到目标数量后，用 `cv2.calibrateCamera` 计算每张图的重投影误差，不合格图移至 `不合格/` 子目录
- **去重策略**：基于角点姿态的三维度比较（中心位置、尺度、归一化形状），替代传统直方图+SSIM

### chessboard_detector.py — 棋盘格检测 & 质量评价

| 方法 | 说明 |
|------|------|
| `detect()` | OpenCV `findChessboardCorners` + `cornerSubPix` 亚像素细化 |
| `check_sharpness()` | 拉普拉斯算子方差 ≥80 判定清晰 |
| `check_exposure()` | 过暗(<20)和过亮(>235)像素均 ≤15% 判定正常 |
| `check_boundary()` | 所有角点距图像边缘 ≥10px 判定不越界 |
| `compute_chessboard_area_ratio()` | 凸包面积/图像面积，<10% 舍弃 |
| `filter_by_reprojection_error()` | 标定→投影→计算 RMS 误差，≤1.0px 为合格 |

### rtsp_capture.py — RTSP 流捕获

后台独立线程持续读取最新帧，解决 `cv2.VideoCapture` 缓冲区堆积问题。`capture_frame()` 线程安全返回帧副本。

### voice.py — 语音播报

后台线程通过 Windows SAPI TTS 播报状态，避免阻塞 GUI。异步队列模型，非阻塞 Speak 调用。

### osd_control.py — OSD 控制

- **ONVIF**：`GetOSDs` / `SetOSD` SOAP 协议，密码 Digest 认证
- **HTTP 直连**：海康威视 ISAPI、大华 CGI 协议
- **备份恢复**：清除 OSD 前备份原始状态，恢复时还原

## 截图流程

```
RTSP 帧 → 棋盘格检测
  ├─ 未检测到 → 日志 + 语音提示
  └─ 检测到角点 →
      ├─ ① 清晰度检测 ─── 模糊？→ 舍弃
      ├─ ② 曝光检测 ───── 过曝/欠曝？→ 舍弃
      ├─ ③ 贴边检测 ───── 越界？→ 舍弃
      ├─ ④ 面积比检测 ─── <10%？→ 舍弃
      ├─ ⑤ 姿态去重 ───── 与已有照片姿态相同？→ 舍弃
      └─ 全部通过 → 保存 JPEG → 缩略图 → 更新预览
```

## 照片质量检测标准

| 检测项 | 方法 | 参数 | 判定 |
|--------|------|------|------|
| 棋盘格角点 | `findChessboardCorners` | — | 未找到全部内角点 → 无效 |
| 清晰度 | Laplacian 方差 | ≥80 | <80 → 模糊 → 无效 |
| 曝光 | 暗/亮像素占比 | 暗<20 / 亮>235, 各≤15% | 超出 → 无效 |
| 贴边 | 角点与边缘距离 | ≥10px | <10px → 越界 → 无效 |
| 面积比 | 凸包面积 / 图像面积 | ≥10% | <10% → 舍弃 |
| 姿态去重 | 角点中心 + 尺度 + 归一化形状 | 中心距>50px 或尺度±15%外 或形状差≥0.05 | 三项均接近 → 重复姿态 → 舍弃 |
| 重投影误差 | `calibrateCamera` + `projectPoints` | ≤1.0px | >1.0px → 不合格 |

### 姿态去重算法

基于棋盘格角点坐标的三维姿态签名：

```python
# 中心位置 — 平移
center = corners.mean(axis=0)
if ||center₁ - center₂|| > 50px → 不同姿态

# 尺度 — 远近
scale = sqrt(mean((corners - center)²))
if scale₁/scale₂ ∉ [0.85, 1.15] → 不同姿态

# 归一化形状 — 旋转/透视
normalized = (corners - center) / scale
if mean(||normalized₁ - normalized₂||²)⁰·⁵ < 0.05 → 相同姿态 → 舍弃
```

优势：即使背景完全相同，只要棋盘格**平移、远近、旋转**任一有显著变化，即判定为有效新图。
